# Rule 06: SEO & AI Discovery

## 1. Meta Titles
- Always use em dash (`—`) as separator: `"Page Title — Vizly"`
- Every route must have a `layout.tsx` with `Metadata` or a `generateMetadata` in its `page.tsx`
- Keep titles under 60 characters for SERPs

## 2. robots.txt (`public/robots.txt`)
- Located at `public/robots.txt`, served at `/robots.txt`
- Allows all crawlers including AI bots (GPTBot, ClaudeBot, PerplexityBot, Google-Extended)
- Blocks `/api/` routes from crawling
- References sitemap at `https://vizly.dev/sitemap.xml`

## 3. llms.txt (`public/llms.txt`)
- Located at `public/llms.txt`, served at `/llms.txt`
- AI-readable overview of the entire site: learning paths, series, articles, guides
- Format: Markdown with sections, each item as `- [Title](/path): Short description`

### REQUIRED: Keep llms.txt up to date
When you add, rename, or remove any of the following, you MUST update `public/llms.txt`:
- A new **guide** (add to the appropriate Guides section)
- A new **article** (add to the Articles section)
- A new **series** (add to the Series section)
- A new **learning path** (add to the Learning Paths section)
- A new **top-level route** (add a new section if appropriate)

### llms.txt format
```markdown
# Vizly — Learn Through Visuals

> Site description.

## Section Name

- [Item Title](/path): Brief one-line description.
```

## 4. Sitemap (`public/sitemap.xml`)
- Auto-generated by `next-sitemap` on every `npm run build`
- Config: `next-sitemap.config.js` in project root
- Gitignored (`/public/sitemap*.xml`) — regenerated each build, never committed
- High priority (`1.0`, `daily`): `/`, `/guides`, `/articles`, `/series`, `/learning-paths`
- Learning path detail pages: `0.9` priority
- Excludes `/api/*` routes
- Referenced from `robots.txt` at `https://vizly.dev/sitemap.xml`
- No manual maintenance needed — new pages are auto-included from the build output
